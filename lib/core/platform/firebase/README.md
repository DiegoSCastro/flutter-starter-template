# Firebase (`core/platform/firebase`)

This directory contains the core services and utilities for initializing and configuring Firebase within the application.

## `FirebaseService`

The central component of this module is the `FirebaseService`. Its purpose is to act as a centralized wrapper that encapsulates the initialization and global configuration of Firebase, keeping `main.dart` clean.

### Responsibilities

1. **Firebase Initialization**: Calls `Firebase.initializeApp()` using the platform-specific options generated by the FlutterFire CLI (`DefaultFirebaseOptions.currentPlatform`).
2. **Crashlytics Configuration**: Sets up global error handlers using `FirebaseCrashlytics` to ensure both synchronous Flutter framework errors and asynchronous unhandled exceptions are recorded as fatal errors.
3. **Background Messaging**: Registers a top-level `@pragma('vm:entry-point')` handler (`firebaseMessagingBackgroundHandler`) to process incoming Firebase Cloud Messaging (FCM) messages while the app is in the background or terminated.

### Usage

`FirebaseService` is registered as a singleton using `getIt`. It must be initialized as early as possible in `main.dart`, immediately after dependency injection is configured:

```dart
void main() async {
  WidgetsFlutterBinding.ensureInitialized();
  await configureDependencies();

  // Initialize Firebase and related global configurations
  await getIt<FirebaseService>().init();

  // Initialize other services...
  runApp(const App());
}
```

> [!NOTE]
> Other services (like `FirebaseMessagingService` or `FirebaseAnalyticsService`) depend on the underlying `FirebaseApp` instance. Therefore, `FirebaseService.init()` must complete before initializing those services.

## Setup Instructions

To connect this template to your own Firebase project, use the [FlutterFire CLI](https://firebase.flutter.dev/docs/cli/).

1. **Install the CLI**:
   ```bash
   dart pub global activate flutterfire_cli
   ```

2. **Run Configuration**:
   ```bash
   flutterfire configure
   ```
   This command will guide you through selecting your Firebase project and automatically generate the necessary `lib/firebase_options.dart` file for the supported platforms (iOS, Android, Web, macOS).

3. **Verify Settings**: Ensure that the newly generated `firebase_options.dart` is placed at the root of the `lib` folder so that `FirebaseService` can locate `DefaultFirebaseOptions.currentPlatform`.
