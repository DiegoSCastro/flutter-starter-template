# Generated Code (`gen`)

This directory contains auto-generated code for the Flutter application. 

Currently, it houses files generated by [`flutter_gen`](https://pub.dev/packages/flutter_gen) (or similar asset-generation tools), which provides type-safe references to local assets like images, SVGs, and fonts.

> [!WARNING]
> **Do not modify the files in this directory manually!** Any changes made here will be overwritten the next time the generator runs.

## Assets Generation

The file `assets.gen.dart` contains strongly-typed classes that reference the assets defined in the `pubspec.yaml` file (e.g., in the `assets/` directory at the project root). 

Using generated asset classes helps avoid runtime errors caused by typos in asset paths.

### Usage in Code

Instead of hardcoding string paths, use the `Assets` class to load images and SVGs:

```dart
import 'package:flutter_starter_template/gen/assets.gen.dart';

// Loading an SVG icon
Widget buildIcon() {
  return Assets.icons.bookmark.svg(
    width: 24,
    height: 24,
    colorFilter: const ColorFilter.mode(Colors.black, BlendMode.srcIn),
  );
}

// Loading a placeholder/empty state image
Widget buildEmptyState() {
  return Assets.images.emptyState.svg();
}
```

### Regenerating Assets

When you add, rename, or delete assets in the `assets/` folder, you must regenerate the `assets.gen.dart` file. 

Run the following command to update the generated code:

```bash
flutter dart run build_runner build --delete-conflicting-outputs
```
*(Depending on how `flutter_gen` is configured in the project, it might also regenerate automatically via a `pubspec.yaml` hook or a standalone `fluttergen` command.)*
